# TASK: Multi-Agent Graph (gMAS)

## Goal

С нуля спроектировать и реализовать **небольшую мультиагентную систему** на **gMAS** из монорепозитория (`gmas/`): минимум **три роли**, явный граф переходов, один сценарий запуска (CLI или `python -m …`), конфигурация LLM только через **переменные окружения**, **`.env.example`** без секретов, **pytest** с моком или заглушкой там, где реальный LLM в CI недоступен.

Итог — **рабочий код в этом каталоге**, а не описание «на бумаге». Документация в `README`: как установить зависимости, как запустить, какие переменные нужны.

## Repository context

- **gMAS** — каталог `gmas/` в корне репозитория; менять его только при необходимости и отдельном согласовании, основная работа — здесь, в `workspaces/multi_agent_gmas/`.
- При работе из Umbrella с доменом **`multi_agent_gmas`**: перед существенными правками в воркспейсе использовать контекст GMAS (`get_gmas_context`, `search_gmas_knowledge`), как требует политика навыка.

## Functional requirements

1. **Предметная область** — на выбор исполнителя (исследование, планирование, разбор документа и т.п.), зафиксировать в `README`; главное — **≥ 3 осмысленных агента** и **явная передача артефактов** между узлами графа.
2. **Сборка графа** через публичный API gMAS (`GraphBuilder`, роли, рёбра или эквивалент по актуальной документации в `gmas/docs`).
3. **Обработка ошибок** на уровне оркестрации: понятное сообщение при сбое узла, таймауте или пустом вводе; лог или трассировка шагов для отладки.
4. **Тесты**: хотя бы unit на разбор/валидацию входа и smoke с подменой вызова LLM/раннера.

## Non-functional requirements

- Зависимости зафиксированы в `pyproject.toml` или `requirements.txt` после появления кода.
- Запуск из корня этого воркспейса **одной–двумя командами** (указать в `README`).

## Acceptance criteria

- [ ] Реализован граф **≥ 3 агентов** на gMAS с осмысленными ролями.
- [ ] Есть точка входа и **README** с архитектурой и запуском.
- [ ] Есть **`.env.example`**; секретов в репозитории нет.
- [ ] Есть **автотесты** и описание их прогона в `README`.

## Recommended workflow

1. Обзор `gmas/docs` и примеров в `gmas/examples/`.
2. Чертёж графа и контрактов данных между узлами.
3. Код, точка входа, тесты.
4. Проверка чеклиста приёмки.
