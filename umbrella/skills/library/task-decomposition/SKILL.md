## Decomposition

Break the task into independently verifiable slices.

Each slice should name:
- Goal.
- Files or modules expected to change.
- Dependencies on other slices.
- Verification command or observation.
- Acceptance criteria.

Keep the umbrella phase plan separate from any internal implementation checklist.
